Project 01 · 个人主页

个人主页承担的是“把分散的兴趣和身份入口收束到一个页面”的角色。它不是单纯的名片页，而是把游戏资料、追剧清单、Bilibili 入口、头像和常用链接放进一个响应式界面里。

新版截图显示，页面已经从早期毛坯状态进入更完整的视觉层：信息块更清楚，游戏和剧集入口具有卡片化组织，移动端也能保留主要信息层级。

Project 02 · 森林卫星图像识别

目标

降低普通用户理解遥感影像的门槛：用户在地图上选择区域后，程序自动获取近期可用影像，计算森林覆盖、有效像素、变化趋势和可解释建议。

流程

区域选择、NASA GIBS 请求、影像预览、像素统计、历史对比和文字化判断被串成一个桌面端工作流。

边界

项目适合辅助观察森林覆盖变化，但不替代专业灾害预警、气象判断或林业部门的正式监测数据。

Project 03 · 暗号频道

产品定位

“暗号频道”是一个赛博风、异世界主题聊天室。访客不能直接进入频道，需要填写代号、选择头像、提交申请，并等待管理员审核通过。

系统能力

它覆盖公聊、私聊、SSE 实时更新、轮询回退、用户状态管理、消息软撤回、头像库管理、管理员限流登录和 Docker 部署。

能力演进路径

暗号频道架构与安全要点

层级 实现内容 承担职责
前端 原生 HTML / CSS / JavaScript 申请、聊天、私聊、头像选择和内置管理入口。
后端 FastAPI / Uvicorn API、认证、静态资源、事件推送和管理员操作。
数据 SQLite 保存用户、消息、私聊会话、清屏起点和管理状态。
实时 SSE + 轮询回退 兼容公网隧道和长连接不稳定场景。
安全 CSP / 登录限流 / 令牌过期 / HMAC IP 指纹 降低脚本注入、暴力登录、隐私泄露和越权管理风险。

后续优化建议

个人主页

补强内容维护和可访问性

增加更新记录、链接有效性检查、移动端细节校验和无障碍文本，让展示页更像长期维护的作品档案。

森林卫星图像识别

提高分析可信度

补充更多样本区域、输出置信区间说明、形成报告导出格式，并用固定测试集验证结果稳定性。

暗号频道

完善运维与恢复能力

补齐部署说明、自动化流程测试、备份恢复演练和生产配置清单，让项目从“可运行”进一步走向“可维护”。

结论

这三个项目共同展示了一条清晰的成长路线：先把信息组织成可浏览页面，再把外部数据转化为分析结果，最后进入多用户、实时、安全和持久化的 Web 系统。

从作品集角度看，它们不是彼此孤立的练习，而是逐步扩展技术边界的组合：页面表达、数据处理、服务端设计和协作体验都已经有可展示的成果。下一阶段最重要的不是堆功能，而是提高验证、文档、部署和长期维护的质量。
